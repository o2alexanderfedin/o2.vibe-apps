## Task 1 — `registryKey(kind, type, prompt?)` (TDD: red → green)
- `src/registry/cacheKey.ts`: extracted `normalizePart` (NFC + lowercase + trim +
  collapse-ws) and `sha256Hex`; **`cacheKey(input)` is byte-identical** (its 7
  tests pass unchanged — kept as the single-string opaque primitive). Added
  `RegistryKind = "app"|"widget"|"handler"` and `registryKey(kind, type, prompt="")`
  = SHA-256 over `[kind, normalizePart(type), normalizePart(prompt)]` joined by a
  `U+001F` unit separator (`String.fromCharCode(0x1f)` — visible in source, not an
  invisible control char). The separator is non-whitespace so it survives
  normalization → field boundaries can't blur.
- `src/registry/cacheKey.test.ts`: +10 tests (wrote first, confirmed RED, then
  GREEN) — determinism, kind-distinctness (app≠widget≠handler same slug),
  prompt-distinctness, prompt+type normalization equivalence, no field-boundary
  blur, absent-prompt = empty-prompt, 64-hex opacity, unicode/emoji-safe.
