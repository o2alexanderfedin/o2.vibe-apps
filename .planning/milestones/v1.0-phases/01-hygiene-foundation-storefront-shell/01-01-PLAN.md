<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ↔ persistent storage | `localStorage` (config) and IndexedDB (registry) hold user data; private-mode/zero-quota can make IndexedDB unavailable. |
| document load ↔ network | The CSP meta tag defines the only allowed outbound destinations; no egress is made in Phase 1 but the policy is set now. |
| application code ↔ devtools surface | Symbol names, store names, console output, and CSS are all F12-visible; sourcemaps would expose comments/identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | production bundle (sourcemaps) | mitigate | `build.sourcemap: false` in `vite.config.ts` (D-04/D-45); acceptance asserts no `.map` in `dist/`; never toggled true in CI. |
| T-01-02 | Information Disclosure | console logging | mitigate | All logging funnels through `src/lib/logger.ts`, off unless `localStorage.debug` set at load (D-30); neutral copy only (D-32); API key never passed in (D-13/D-37). |
| T-01-03 | Denial of Service | IndexedDB (private mode / zero quota) | mitigate | Probe write/delete at init detects unavailability; degrade to in-memory `Map` with identical async interface so the app keeps working (D-22/D-23). |
| T-01-04 | Information Disclosure | network egress destinations | mitigate | CSP `connect-src 'self' https://api.anthropic.com` set in `index.html` now (D-43/SEC-04), even though no fetch occurs in Phase 1 — locks the final policy. |
| T-01-05 | Tampering | cross-origin script/data injection | mitigate | CSP `default-src 'self'`; `script-src 'self' 'unsafe-eval'` (eval scoped for the future compile path only, Pitfall 5); no `innerHTML` in the shell. |
| T-01-06 | Information Disclosure | mechanic disclosure via neutral-naming slip | accept (covered in Plan 04) | Lexicon gate (`src/hygiene.test.ts`) in Plan 04 scans all source produced here; this plan uses neutral identifiers/store names by construction. |
</threat_model>
