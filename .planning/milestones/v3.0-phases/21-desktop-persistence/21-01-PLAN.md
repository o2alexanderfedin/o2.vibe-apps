<objective>
Establish the two foundation primitives Phase 21 depends on: the raw-key settings store seam and the pure layout persistence module.

Purpose: DesktopShell will write/read the persisted layout via `services.settingsStore.writeRaw`/`readRaw` — these methods must exist on the interface and the production implementation before the save and restore effects can be wired. The `layoutPersistence.ts` module provides the single source of truth for the IDB key constant, the exact LayoutEntry shape, and the serialize/deserialize functions so no layout code is scattered across DesktopShell.

Output: Extended SettingsStore interface, extended realSettingsStore implementation, updated RecordingSettingsStore test double, new layoutPersistence module with unit tests.
</objective>
