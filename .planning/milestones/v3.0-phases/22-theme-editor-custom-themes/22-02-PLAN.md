contrastRatio context (from 22-RESEARCH.md §8):
  - Pure function, no browser API, safe in JSDOM
  - WCAG 2.1 relative luminance: linearize each channel (c <= 0.04045 ? c/12.92 : ((c+0.055)/1.055)^2.4); luminance = 0.2126*R + 0.7152*G + 0.0722*B
  - contrastRatio: (max(L1,L2)+0.05) / (min(L1,L2)+0.05)
  - Returns null if either value is not a parseable hex color (#rgb or #rrggbb)
  - Returns null for rgba/gradient strings (parseHex returns null)
  - ThemeEditor calls contrastRatio("--text" value, "--b1" value) to show the advisory warning when ratio < 4.5

Known pairs for tests (from WCAG spec):
  - contrastRatio("#ffffff", "#000000") === 21.0 (black on white)
  - contrastRatio("#000000", "#ffffff") === 21.0 (white on black — symmetric)
  - contrastRatio("#777777", "#ffffff") ≈ 4.48 (just below AA threshold of 4.5)
  - contrastRatio("rgba(255,255,255,0.10)", "#000000") === null (non-hex fg → null)
  - contrastRatio("#ffffff", "radial-gradient(...)") === null (non-hex bg → null)
  - contrastRatio("#f3f1ff", "#7c5cff"): both hex, returns a number (use toBeGreaterThan(1))
