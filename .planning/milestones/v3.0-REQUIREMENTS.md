# Requirements: Vibe App Store тАФ Milestone v3.0 "Trusted Desktop"

**Defined:** 2026-06-26
**Core Value:** A user opens an app from the storefront and it renders and works тАФ instantly on a cache hit, seamlessly produced on a cache miss тАФ and nothing visible ever reveals that the app was made on demand.

**Milestone goal:** Make the Vibe OS desktop *safe to run untrusted generated code* (real opaque-origin `<iframe sandbox>` isolation, the key never in the frame), *durable across reloads* (window layout + theme restored), and *personalizable* (a custom-theme editor) тАФ without breaking the illusion that apps simply exist.

> **Build order is dependency-enforced** (confirmed independently by all four research streams): **CHROME тЖТ SANDBOX тЖТ PERSIST / THEME-editor**. The `тЛо`-to-titlebar relocation (CHROME-01) is a *hard prerequisite* for the iframe work тАФ once an app body is an opaque frame, no host chrome can be injected into it (`createPortal` across the boundary needs `allow-same-origin`, which must never be set).

## v3.0 Requirements

### CHROME тАФ Window chrome & UX

> Host-owned window controls. Relocating the contextual menu into the titlebar is the prerequisite that lets the app body become an opaque frame.

- [ ] **CHROME-01**: The per-app `тЛо` contextual menu (the MOD-01 prompt) is **relocated into the `WindowFrame` titlebar, right-aligned** (opposite the traffic-lights); `ContextualPrompt` renders from the frame, and the now-redundant in-body app-shell header is removed. Remove / clone / tweak (MOD-01..04) all still work, triggered from the titlebar. *(Hard prerequisite for all SANDBOX work.)*
- [ ] **CHROME-02**: A window can be **maximized / unmaximized** тАФ it zooms to fill the **work area** (between the menu bar and the dock), **not** OS full-screen; double-clicking the titlebar toggles it; unmaximize restores the prior geometry.
- [ ] **CHROME-03**: A window **snaps to the left or right half** тАФ by dragging to the screen edge (with a translucent drop-zone preview) and via `Ctrl+Left` / `Ctrl+Right`.
- [ ] **CHROME-04**: **Keyboard shortcuts** act on the active window тАФ `Cmd/Ctrl+W` closes, `Cmd/Ctrl+M` minimizes тАФ each with `preventDefault` so the browser tab is never closed.

### SANDBOX тАФ Untrusted-code isolation (HARD-01)

> The flagship. Convert containment-by-convention (`new Function`) into a genuine opaque-origin boundary. Every key-leak pitfall concentrates here; each requirement is CI-enforced, not review-only.

- [ ] **SANDBOX-01**: Each app body renders inside an **`<iframe sandbox="allow-scripts">`** with an **opaque origin** (`allow-same-origin` is never set); the compiled component instantiates **inside the frame** with the frame's own inlined React copy (via `srcdoc`). A CI test asserts the mounted `sandbox` attribute never contains `allow-same-origin`.
- [ ] **SANDBOX-02**: The **Anthropic API key never enters the frame**. The srcdoc builder is type-enforced (`buildSrcdoc(transpiledJS, themeVars, parentOrigin)` тАФ no other params); a CI test asserts the srcdoc string carries no key pattern; `localStorage` is **inaccessible** (`SecurityError`) from inside the frame.
- [ ] **SANDBOX-03**: A typed **`postMessage` RPC** brokers every appтЖФhost call (`runHandler`, `fetchData`, contextual modify) with `crypto.randomUUID()` correlation IDs and **both** `origin === "null"` **and** `source === frame.contentWindow` validation; forged messages from unknown sources are dropped; the **`dataBroker` allowlist is enforced parent-side** on the frame's behalf.
- [ ] **SANDBOX-04**: **Theme CSS vars reach every frame** тАФ injected into the srcdoc `<style>` at construction and **re-pushed on every theme switch** (`THEME_PUSH` broadcast to all live frames) so frames re-skin live in lockstep with the host (CSS custom properties do not cross the iframe boundary).
- [ ] **SANDBOX-05**: An **in-tree fallback mode** (injected via `ServicesProvider`) keeps the existing **727 RTL/JSDOM tests green** without a real browser; at least one **Playwright / browser-native integration test** proves the real frame round-trip тАФ renders, theme-in-frame applies, `localStorage` throws `SecurityError`, and a forged `postMessage` is dropped.
- [ ] **SANDBOX-06**: A frame that **stops responding** (e.g. an infinite loop) is detected via ping/timeout and surfaced as an **unresponsive-app overlay with force-close**; the rest of the desktop stays usable. *(Known limitation: the loop itself cannot be `terminate()`d without a Worker тАФ this is contained, not solved.)*

### PERSIST тАФ Desktop persistence

> Restore the desktop the user left. Additive keys in the existing IDB `settings` store тАФ no DB version bump, no migration.

- [ ] **PERSIST-01**: The **window layout** persists across reloads тАФ per window `{ appType, title, icon, x, y, z, minimized }` (plus the open-app set), written to the existing IDB `settings` store under a new key, via a **debounced** write on any geometry change (no per-pixel write storm).
- [ ] **PERSIST-02**: On boot, the desktop **restores saved windows** тАФ minting fresh `instanceId`s, restoring geometry/z-order/minimized, and re-resolving each component through the **cache-hit path**; restores are **serialized** (concurrency-capped) so the produce gate isn't thrown.
- [ ] **PERSIST-03**: An app that can **no longer be resolved** on restore (evicted from cache / no key) opens as a **placeholder with a retry action** тАФ never silent API spend; **no in-app state** (scroll, form values) is persisted (generated apps have no stable serialization contract).

### THEME тАФ Theme editor & custom themes (continues THEME-01..05)

> Personalize over the 12-var contract. Dirt-simple by design; the built-in four stay read-only.

- [ ] **THEME-06**: A **theme editor** (opened from the menu bar) edits the **12-variable contract** with native color inputs and **live preview** (mutating `:root` in real time); an invalid color value is **rejected** (`CSS.supports` gate) before any save.
- [ ] **THEME-07**: Custom themes can be **named, saved, duplicated-from-a-built-in, and deleted**, persisted in the IDB `settings` store under a `custom:<name>` namespace; names are **sanitized** (`sanitizeDisplayName`) and namespaced so they **cannot collide** with the built-in four (which remain read-only). Deleting the active theme auto-switches to Aurora first.
- [ ] **THEME-08**: A saved custom theme appears in the **menu-bar switcher** alongside the built-ins, and selecting it **re-skins the host AND all open frames live** (`THEME_PUSH`), exactly like a built-in.
- [ ] **THEME-09**: A custom theme **survives reload FOUC-free** тАФ its vars are mirrored to `localStorage` at save time, and the `index.html` FOUC script applies the active custom theme on first paint (with the `csp.test.ts` SHA-256 hash updated in the **same commit**).
- [ ] **THEME-10**: The editor shows an **inline, non-blocking WCAG contrast warning** when a text/background pairing would be hard to read.

### HYGIENE тАФ Illusion enforcement (continues HYGIENE-01..06)

- [ ] **HYGIENE-07**: The CI lexicon gate is **extended to all new v3.0 surfaces** тАФ `frameBridge.ts`, `SandboxFrame.tsx`, `ThemeEditor.tsx`, the **srcdoc template constant**, `postMessage` payload field names, and new IDB keys тАФ so neither the mechanic lexicon **nor** the words "iframe / sandbox / isolation" leak into any devtools-visible surface or UI copy.

## Cross-Cutting Constraints (acceptance criteria on EVERY phase тАФ not separate phases)

These are inherited from v1.0тАУv2.0 and gate every v3.0 phase:

- **Zero new npm runtime dependencies** тАФ all four pillars confirmed dep-free (Playwright as a *dev*-dependency for SANDBOX-05 is the one allowed exception, outside the runtime-bundle bias).
- **Key never enters the frame** тАФ type-enforced `buildSrcdoc`; CI-tested.
- **Devtools-hygiene lexicon gate** green across `src/**` + `index.html` + srcdoc strings + postMessage field names.
- **IoC / DI** with captured-Haiku fixtures; new components reach the suite via the `in-tree` fallback mode.
- **`build.sourcemap: false`** + minify in prod; **CSP** `connect-src 'self' api.anthropic.com` (the frame never calls Anthropic directly).
- **Additive IDB only** тАФ new keys in the existing `settings` store; no DB version bump.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| **OS-level full-screen** (hides dock + menu bar) | Destroys the desktop chrome that *is* the product identity; maximize = zoom-to-work-area instead. Permanent exclusion, not a deferral. |
| **Snap Assist cascade** (auto-fill the other half) | Significant state complexity for marginal real-world use; manual half-snap suffices. |
| **Color-theory / HSL wheel / palette generation** in the editor | Obscures the 12-var semantic contract; direct labeled pickers are clearer. |
| **Community theme gallery** | Requires a server тЖТ violates the zero-infra, client-only constraint. JSON export/import (v3.1) is the sharing primitive. |
| **Per-window themes** | The theme contract is global; per-frame overrides break OS aesthetic coherence. |
| **Undo/redo in the theme editor** | High cost for a 12-var editor; duplicate-before-edit substitutes. |
| **Persisting in-app state** (scroll, form values) | Generated apps have no stable serialization contract; stale state confuses users. |
| **Worker-based termination of a runaway frame** | An iframe can't be `terminate()`d; SANDBOX-06 contains the impact (overlay + force-close) rather than pretending to kill the loop. |
| **Naming the mechanic** *(AI / LLM / generate / synthesize)* **OR the words "iframe / sandbox / isolation"** in any user- or devtools-visible surface | The illusion is the product; enforced by the CI lexicon gate (HYGIENE-07). |
| **Proxying the Anthropic API / any server** | Breaks the client-only, zero-infra, never-proxy-the-key model. |
