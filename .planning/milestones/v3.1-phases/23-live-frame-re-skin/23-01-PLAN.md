<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host -> opaque frame | postMessage THEME_PUSH carries 12 CSS custom property key-value pairs from the host browsing context to the frame's opaque-origin context; the frame cannot reach the host's DOM, localStorage, or API key |
| frame bootstrap handler | The frame-side message listener processes THEME_PUSH and calls document.documentElement.style.setProperty(k, vars[k]) for each entry in the vars object |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | THEME_PUSH handler (frameMount.ts:407-415) — `style.setProperty(k, vars[k])` | accept | CSS custom property injection via style.setProperty is scoped to CSS styling; browsers silently ignore property names that are not valid CSS custom properties (must start with `--`). No script execution is possible via style.setProperty. The vars values are the 12 well-known CSS custom property keys from the host theme system. This fix does not expand the set of vars or the handler's trust surface (both existed since Phase 20). |
| T-23-02 | Information Disclosure | broadcastTheme -> `postMessage({...}, "*")` | accept | targetOrigin `"*"` is unchanged from the Phase 20 decision (opaque-origin frames cannot receive messages targeted to the string `"null"`; `"*"` is the only viable option). The THEME_PUSH payload contains only the 12 CSS custom property values — no PII, no API key, no app code. Any frame on the page could receive the message, but the payload is non-sensitive CSS variables. Accepted in Phase 20; this plan does not change the targetOrigin or the payload. |
| T-23-03 | Denial of Service | srcdoc memo dep array causing silent state loss on theme switch | mitigate (this plan) | Before this fix, every theme switch silently destroyed in-frame React state by forcing an iframe reload. After the fix the iframe is stable across theme changes — this is the intended mitigation. The fix reduces the denial-of-service surface by removing the unintended reload. No new attack surface is introduced. |
</threat_model>
