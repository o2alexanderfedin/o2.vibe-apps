<objective>
Add `openAt` to the window manager so the restore effect can open windows at their exact persisted geometry instead of cascade-placing them.

Purpose: `open()` uses `cascadePlace()` and assigns z via `++zTop`. For restore, both are wrong — the exact persisted x/y/z must be set atomically at open time. `setGeometry` cannot be used after `open()` because the window id is not available until after React re-renders the state update. `openAt` sets all geometry fields in the initial WindowEntry, bypassing cascade entirely, and adjusts `zTop` so future open/focus calls assign correctly higher z values.

Output: `openAt` method in `WindowManagerValue` interface, implemented in `WindowManagerProvider`, exported in the context value object, covered by unit tests.
</objective>
