<objective>
Build `WindowFrame` — the outer draggable glass chrome that wraps the existing `AppShell` (which keeps the contextual-prompt MOD-01..04 wiring as the window's CONTENT). WindowFrame is the macOS-style frame: traffic-light titlebar (close/minimize/decorative), app icon + title, a draggable titlebar handle (wired to `useDrag` from Plan 01), and a body container into which the app mounts via `mountApp` (the `mount.ts` seam). It consumes `WindowEntry` props from `useWindowManager` (Plan 02) and the manager's focus/minimize/close callbacks.

Also add the CSS: `.desktop` (the isolated window-host container, `isolation: isolate`), `.window-chrome` (glass via Phase 14 `--glass`/`--bord` vars + `display:none` when minimized), `.titlebar-handle`, and `.traffic-light` rules — all neutral-named.

Defends Pitfalls 1/2 (drag via useDrag), 3 (window lives in the isolation:isolate `.desktop`), 4 (minimized = display:none, no compositor layer), 8 (cleanup calls unmountApp), 9 (body-mount guarded by document.contains), 12 (raise-to-front on body pointerdown does NOT preventDefault, so app `<input>` keeps focus).

Purpose: An opened app renders inside a draggable glass window with shared chrome (WIN-01); pointer-down raises it (WIN-03); minimize hides it via display:none (WIN-04); the frame's lifecycle ties to the React-root teardown (WIN-05).
Output: `src/ui/WindowFrame.tsx` + `src/ui/WindowFrame.test.tsx` + CSS additions in `src/index.css`.
</objective>
