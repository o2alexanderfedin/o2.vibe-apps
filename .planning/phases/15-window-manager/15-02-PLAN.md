<objective>
Build `useWindowManager` — the single source of window truth. It **mints the `instanceId`** (CONTEXT Decision 3 — the manager is the one source keyed on instanceId; Marketplace MUST NOT mint its own), owns `WindowEntry[]` React state in parallel with `mount.ts`'s roots Map (keyed on that same `instanceId`), a module-level `zTop` counter (starting at 200), and the open/focus/minimize/restore/close API plus an `isOpen(id)` guard. This is self-contained logic with a clean analog (`VibeThemeProvider.tsx`) and ships in parallel with `useDrag`.

It defends Research Pitfalls 3 (bounded zTop counter, not arbitrary large numbers), 4 (minimize is state-only — display:none is applied by WindowFrame in Plan 03), 8 (close MUST call unmountApp — the central leak guard), and 9 (mid-produce-close: a closed window's later mount must be a no-op so no orphan root leaks — `isOpen(id)` is the PRIMARY guard for this, checked by the open flow before mounting; WindowFrame's `document.contains(container)` check is the last-resort backstop).

Purpose: Multiple apps open concurrently as independent windows (WIN-02), each raised/focused on demand with cascade placement (WIN-03), minimizable/restorable (WIN-04), and closed with zero React-root leak (WIN-05) — verified by `mountedCount()` returning to baseline.
Output: `src/ui/useWindowManager.tsx` (hook + provider + context + WindowEntry type) + `src/ui/useWindowManager.test.tsx`.
</objective>
