<objective>
Rewire the open flow from the old inline `openedApps`/`AppShell` region to the windowed model. `App.tsx` gains a `WindowManagerProvider` wrapper and renders the `.desktop` container (one `WindowFrame` per `WindowEntry`). `Marketplace.tsx` card clicks call `useWindowManager.open(...)` instead of pushing to local `openedApps` state — but the resolve→produce→cache→mount path (`resolveOpenApp`, `resolveComponent`, `evictLiveComponent`) and the ContextualPrompt MOD-01..04 routing stay intact (only the mount TARGET changes to a WindowFrame body).

SINGLE SOURCE OF instanceId (CONTEXT Decision 3 — resolves the checker's double-source warning): the manager mints the instanceId. `useWindowManager.open(appType, meta)` RETURNS the minted instanceId; `handleOpen` consumes THAT id for `resolveComponent` and for keying its Component map. Marketplace's old `nextInstanceId` is REMOVED from the open path — there is exactly one mint, in the manager.

SETTLED MOUNT CONTRACT (from Plan 03 — resolves the checker's fork): the app is NOT a host-tree child of WindowFrame. WindowFrame mounts, via `mountApp(instanceId, bodyRef, Wrapper)`, a Wrapper that renders `<AppShell ...>{<Component/>}</AppShell>` as a SINGLE root. So WindowFrame already owns the AppShell + ContextualPrompt + Component lifecycle; this plan only supplies the props (Component, onClose, onModify, onFocus, onMinimize, onMove) and the `.desktop` host. Do not re-introduce an inline AppShell render here.

Then ship the end-to-end integration test suite — the six mandated Phase-15 behavior tests driven through the REAL rendered open flow with injected canned services (no live network), mirroring the existing `Marketplace.test.tsx` harness.

This is the HIGHEST regression-risk plan: it touches the render/open path and must keep all existing MOD/AppShell/Marketplace tests green. Defends Pitfalls 3 (cascade + bounded z), 8 (close zero-leak), 9 (mid-produce-close), 12 (focus-not-stolen) at the integration level.
