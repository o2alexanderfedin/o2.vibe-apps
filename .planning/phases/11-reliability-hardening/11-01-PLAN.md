<objective>
Add `zod` as a production dependency, create the `deriveStateSchema` helper that derives a lenient-partial zod/mini schema from `module.initialState`, and wire it into the `DelegatedShell` merge step so a mis-typed known field is rejected and prior state is kept.

Purpose: This is the core of RELY-01 and RELY-03 — the merge step that currently blindly spreads `next` into state will now guard against corruption from produced handlers that return a known field with the wrong type. The validation is lenient by design (partial, passthrough for unknown keys) to honor the reliability paradox.

|----------|-------------|
| produced handler → DelegatedShell state | Handler output (untrusted, produced by LLM) crosses into React state via the merge step — the new validation boundary |
| initialState schema derivation | The schema is built from developer-controlled initialState, not from handler output — this is a trusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-11-01 | Tampering | DelegatedShell merge step | mitigate | Schema validation at merge step (this plan) — a produced handler that returns a known field with a wrong type is rejected; prior state is kept |
| T-11-02 | Denial of Service | safeParse call per click | accept | safeParse is synchronous, O(1) for flat objects — no blocking, no additional latency beyond the existing runHandler await |
