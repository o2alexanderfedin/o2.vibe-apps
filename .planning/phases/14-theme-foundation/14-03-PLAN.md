<objective>
Build `VibeThemeProvider` — the named-theme engine that owns the four-theme CSS-variable contract on `document.documentElement` — layered on top of the existing `ThemeProvider` (which keeps the light/dark/system data-theme mechanism intact). Define the `VIBE_THEMES` constant verbatim from the design, apply variables via `style.setProperty`, persist to localStorage, and mirror to the IDB `settings` store through an injected IoC/DI seam so tests run fully offline.

Purpose: This is the central, load-bearing theming mechanism (THEME-02 live re-skin, THEME-04 document-root application + dual persistence). Applying on documentElement (not a React context value) is what lets CSS variables cascade into every separately-createRoot'd generated-app subtree (Pitfall 5). The existing 552 tests depend on ThemeProvider being untouched — this LAYERS, it does not replace.
Output: `VibeThemeProvider.tsx` (+ VIBE_THEMES), the `settingsStore.ts` seam wired into Services, the provider mounted in App.tsx below ThemeProvider, and a TDD test proving the behavior offline.
</objective>
