Add the backward-compatibility alias bridge so apps cached before v2.0 keep rendering after the new theme variables land, plus the neutral localStorage key constant for the active OS theme name.

Output: An always-on `:root` alias rule in src/index.css; `STORAGE_KEY_OS_THEME` in storage.ts; a test pinning the mapping.
