<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated handler → fetchData closure | Generated code can only call fetchData(sourceId, params); it never sees the Services object, the broker implementation, or the Anthropic key |
| fetchData closure → DataFetchBroker | The closure is built in loader/handler host code; it guards the Services object behind the closure boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-A | Elevation of Privilege | Generated handler accessing Services | mitigate | boundFetchData is a closure that reads services.fetchDataBroker inside host code; Services is never passed as a parameter into the generated scope |
| T-12-03-B | Elevation of Privilege | Generated handler bypassing broker via raw fetch | mitigate | "fetch" and "XMLHttpRequest" stay in DENIED_GLOBALS; they remain shadowed to undefined; fetchData is the only network-capable symbol in handler scope |
| T-12-03-C | Denial of Service | Missing broker crashing the handler | mitigate | boundFetchData uses ?. guard + ?? Promise.resolve({error}) fallback — absent broker returns neutral error, never throws, never blocks the core loop |
| T-12-03-D | Info Disclosure | API key entering data path | accept | The data path is keyless; createDataBroker does not accept or use a key; the Anthropic key is only on the model-transport path (Services.getApiKey → transport) which fetchData does not touch |
</threat_model>
