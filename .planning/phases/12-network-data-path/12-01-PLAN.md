<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| handler caller → broker | Generated handler code calls fetchData(sourceId, params); only sourceId + params cross the boundary |
| broker → external API | The broker (not generated code) builds and sends the HTTP request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01-A | Spoofing | DataFetchBroker sourceId lookup | mitigate | SOURCE_MANIFEST.get(sourceId) — only exact manifest keys resolve to an origin; unknown keys return {error} immediately with no fetch |
| T-12-01-B | Tampering | URL parameter injection via params | mitigate | Only allowedParams-declared keys from SourceManifestEntry are encoded into URLSearchParams; all other keys silently dropped at the broker |
| T-12-01-C | Info Disclosure | Neutral error copy | mitigate | All error paths return NEUTRAL_DATA_ERROR constant — no HTTP status code, no origin name, no provider name, no mechanic token surfaced |
| T-12-01-D | Denial of Service | Unbounded data fetches | mitigate | TokenBucket limiter wraps every data fetch with the same rate/concurrency contract as the model transport; 429 responses trigger computeBackoffDelay retry |
| T-12-01-E | Elevation of Privilege | Anthropic key in data path | accept | The data path is keyless — no key parameter, no Authorization header; the Anthropic key is solely on the model-transport path which this broker does not touch |
</threat_model>
