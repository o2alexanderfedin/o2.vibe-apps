**Accent (`--color-accent-primary`) is reserved for exactly these elements:**
1. The primary submit button in KeyDialog ("Connect account" / "Save" CTA)
2. The active/selected state indicator on the 3-way theme toggle
3. Card hover border highlight (border transitions from `--color-border-secondary` to `--color-accent-primary`)
4. Focus ring on all interactive elements (`outline` color)

Accent is NOT used for: card backgrounds, AppBar background, dialog backdrop, body text, icon fill (icons use `--color-text-secondary`).

### Theme Application

- Attribute: `data-theme="light"` or `data-theme="dark"` on `<html>` element
- Variables defined on `:root[data-theme="light"]` and `:root[data-theme="dark"]`
- System mode reads `window.matchMedia("(prefers-color-scheme: dark)")` and applies the matching theme attribute
- FOUC prevention: blocking `<script>` in `index.html` before `</head>` reads `localStorage.getItem("marketplace.theme")` and sets `data-theme` synchronously before React mounts
- Source: CONTEXT.md D-15, D-16, D-17, D-18

## FOUC Prevention Script

This exact script (verbatim) belongs inside `index.html` as an inline `<script>` tag placed immediately before `</head>`:

```html
<script>
  (function () {
    var stored = localStorage.getItem('marketplace.theme');
    var theme = stored === 'light' || stored === 'dark' ? stored
      : window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
    document.documentElement.setAttribute('data-theme', theme);
  })();
</script>
```

This runs synchronously before React mounts, preventing any flash of wrong theme. Source: CONTEXT.md D-18.
