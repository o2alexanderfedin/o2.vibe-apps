<objective>
Wire the popular row into the Marketplace storefront: load app records on mount, rank via rankPopular (which owns the useCount >= 1 membership filter), render a conditionally visible section using existing .storefront-grid / .app-card classes, then verify the storefront visually via screenshot.

Purpose: Delivers the user-visible halves of STORE-01 (displayName on cards) and STORE-02 (popular row with honest local-only copy). The popular row reads from persisted records — making it the first record-driven section of the storefront.

Cold-start-hidden contract: the component does NOT apply its own useCount filter. It derives the list ONLY from rankPopular(records) and shows the row iff the returned array is non-empty. Because rankPopular drops every record with useCount < 1, the row is automatically hidden on cold start. This keeps a single authoritative owner of the membership rule (rankPopular in marketplaceUtils.ts, defined in Plan 09-01, tested in Plan 09-02).

Output: Extended Marketplace.tsx with popular row + visual checkpoint confirmation.
</objective>
