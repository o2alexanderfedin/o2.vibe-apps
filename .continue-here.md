## Key new architecture seams (for future work)
- **Theme:** `src/ui/VibeThemeProvider.tsx` + `VIBE_THEMES` apply CSS vars to **`document.documentElement`** (only way they reach separately-mounted generated subtrees). Persist: `localStorage marketplace.osTheme` (FOUC truth) + IDB `settings` store (DB v3). `:root` alias bridge maps `--color-*` → new contract. FOUC script in `index.html` (SHA-256 guarded by `csp.test.ts` — same-commit invariant).
- **Windows:** `src/ui/useWindowManager.tsx` (WindowEntry[] + `zTop`), `useDrag.ts`, `WindowFrame.tsx` (wraps `AppShell`; `hideClose` prop kills the redundant inner ×). Windows render **in-tree** (React subtrees), leak-safe via subtree unmount + `appBodyCount()`.
- **Desktop:** `src/ui/DesktopShell.tsx` (new root), `Dock.tsx`, `MenuBar.tsx`, `iconForApp.tsx`. `Marketplace.tsx` removed.
- **Launcher/produce:** `src/ui/SearchLauncherPanel.tsx` + `launcherUtils.ts` (`slugFromText`); describe→`registryKey("app",slug,text)`→`resolveComponent(...,userPrompt=text)`.
- **Theme-aware gen:** `src/execution/colorCheck.ts` (flags saturated colors, ALLOWS neutral `rgba(0,0,0,α)` shadows — too-greedy fights box-shadows / reliability paradox), wired into the self-heal loop in `producer.ts`; `src/ui/sanitizeDisplayName.ts` at the display boundary.

## Process notes / lessons
- **Verify UI by VIEWING screenshots, not `getComputedStyle`.** User flagged this mid-run: var reads said the theme "worked" while the page showed no colors (legacy storefront didn't consume the vars; the visible re-skin only landed in P16 with the wallpaper). [[verify-ui-visually]]
- The autonomous delegate pattern (plan delegate, then execute+review delegate, both background, sonnet) kept the orchestrator lean across all 5 phases; independent gate re-runs + live smokes caught the panel-CSS gap (via the milestone audit) and were the real quality backstop.
- The pre-push git hook **misfires on TAG pushes** ("tests failed" when the suite is green) — push tags with `--no-verify` AFTER an independent full-suite run.
